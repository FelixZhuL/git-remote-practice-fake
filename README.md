## 任务问题回答
1. 本地仓库和远程仓库分别有什么作用？
答：本地仓库用于个人开发、编辑文件、保存本地版本历史；远程仓库用于代码备份、团队成员共享协作、统一版本管理。
2. 为什么需要把本地提交同步到远程仓库？
答：防止本地文件丢失，方便团队协作开发，统一项目版本，实现多端同步。
3. 远程平台直接修改文件后，本地仓库为什么需要同步远程变化？
答：避免本地与远程仓库版本不一致，同步后才能保持代码统一。
4. 如果本地和远程同时修改了同一个文件，可能会发生什么？
答：会产生Git合并冲突，需要手动解决冲突后才能完成同步。
5. 本次任务中最后一次提交为什么要只包含README.md的回答内容？
答：满足作业规范，保证提交内容干净清晰，区分问答内容与之前的文件修改，便于查看版本历史。
